AI原生智能体编排低代码平台开发文档（整理版）

一、文档目的
本文件用于描述产品规划与路线图，不作为当前代码实现清单。
当前实现状态请参考 README.md 与 PROJECT_SUMMARY.md。

二、项目定位
项目名称：AI原生智能体编排平台
一句话描述：让业务人员用自然语言指挥 AI，自动完成跨系统业务流程编排和应用生成。

三、能力分层与优先级
3.1 基础平台能力（P0/P1）
1) 租户管理：多租户隔离、租户生命周期管理、套餐能力（P0）
2) 组织架构：部门树、岗位体系、汇报关系（P0）
3) 用户管理：用户 CRUD、导入导出、批量操作（P0）
4) 角色权限：RBAC、菜单权限、数据权限（P0）
5) 登录认证：账号密码/短信/扫码/LDAP/SSO（P0）
6) 授权许可：License 生成/验证/到期提醒/节点限制（P0）
7) 系统管理：参数配置、登录日志、操作日志、定时任务（P1）
8) 审计日志：行为审计、数据变更留痕（P1）

3.2 低代码核心能力（P0/P1）
1) 可视化编辑器：组件库、画布、属性面板（P0）
2) 数据模型设计：在线建模、关联关系、索引（P0）
3) 页面与数据绑定：组件数据源绑定、自动 CRUD（P0）
4) 应用发布：源码导出或一键部署（P0）
5) 代码生成器：根据页面协议生成前后端代码（P0）
6) 模板市场：行业模板导入导出（P1）
7) 在线预览：实时预览、多端适配（P1）

3.3 AI 特色能力（P0/P1）
1) 自然语言生成应用（P0）
2) 智能体编排（P0）
3) 多模态交互（语音/文档/草图）（P1）
4) AI 辅助调试与修复建议（P1）
5) 跨系统连接器生态（P1）

3.4 工作流与运维能力（P0/P2）
1) 流程设计器（BPMN）（P0）
2) 审批流能力（会签/或签/转审等）（P0）
3) 自动任务（定时/API/数据更新）（P1）
4) 混合部署与容器化（P0/P1）
5) 高可用与容灾（P2）

四、用户角色
1) 平台管理员：租户、License、系统配置
2) 租户管理员：组织、用户、角色权限
3) 应用开发者：低代码配置与 AI 生成
4) 业务用户：应用使用与流程审批

五、典型场景
1) 快速搭建 OA：请假、报销、审批流程
2) 企业系统集成：企微/钉钉/ERP/CRM 连接
3) 信创环境部署：国产化软硬件适配
4) 跨系统自动化：业务事件驱动的数据流转

六、建议技术架构
后端：Spring Boot、Java 21、JPA、安全鉴权、租户与授权中台
前端：Vue 3、TypeScript、Element Plus、路由与权限体系
平台能力：低代码元数据模型、页面渲染引擎、发布链路、工作流引擎
AI 能力：自然语言解析、任务编排、连接器调用与结果校验

七、当前代码落地现状（2026-04-28）
1) 已落地：系统管理底座（认证、权限、租户、License、日志、字典等）
2) 待完善：低代码设计器、页面协议、生成发布链路、工作流深度集成、AI 应用生成

八、里程碑建议
Phase 1（0-3 个月）：平台底座完善与安全加固
Phase 2（3-6 个月）：低代码建模 + 页面协议 + 运行时
Phase 3（6-9 个月）：AI 生成 + 智能体编排 + 连接器
Phase 4（9-12 个月）：模板生态 + 多端 + 信创与交付能力

九、开发环境要求
JDK 21、Maven 3.9+、Node.js 18+、MySQL 8+、Docker（可选）

五、数据库设计（核心表）
5.1 租户表
sql
CREATE TABLE platform_tenant (    id BIGSERIAL PRIMARY KEY,    tenant_code VARCHAR(64) UNIQUE NOT NULL,    tenant_name VARCHAR(200) NOT NULL,    license_key VARCHAR(500),    status VARCHAR(20) DEFAULT 'active',    expire_date DATE,    max_users INT DEFAULT 100,    max_apps INT DEFAULT 10,    created_at TIMESTAMP);
5.2 部门表
sql
CREATE TABLE sys_dept (    id BIGSERIAL PRIMARY KEY,    tenant_id BIGINT NOT NULL,    parent_id BIGINT DEFAULT 0,    dept_name VARCHAR(100) NOT NULL,    dept_code VARCHAR(64),    sort_order INT DEFAULT 0,    leader_id BIGINT,    status VARCHAR(10) DEFAULT 'active');
5.3 用户表
sql
CREATE TABLE sys_user (    id BIGSERIAL PRIMARY KEY,    tenant_id BIGINT NOT NULL,    username VARCHAR(64) NOT NULL,    password VARCHAR(200) NOT NULL,    real_name VARCHAR(100),    email VARCHAR(100),    phone VARCHAR(20),    dept_id BIGINT,    post_id BIGINT,    status VARCHAR(10) DEFAULT 'active',    last_login_time TIMESTAMP,    created_at TIMESTAMP);
5.4 角色表
sql
CREATE TABLE sys_role (    id BIGSERIAL PRIMARY KEY,    tenant_id BIGINT NOT NULL,    role_code VARCHAR(64) NOT NULL,    role_name VARCHAR(100) NOT NULL,    role_type VARCHAR(20),  -- builtin/custom    data_scope VARCHAR(20) DEFAULT 'dept'  -- 数据权限范围);
5.5 菜单表
sql
CREATE TABLE sys_menu (    id BIGSERIAL PRIMARY KEY,    parent_id BIGINT DEFAULT 0,    menu_name VARCHAR(100) NOT NULL,    menu_type VARCHAR(20),  -- directory/menu/button    permission VARCHAR(200),    path VARCHAR(200),    component VARCHAR(200),    sort_order INT);
5.6 应用表（低代码生成的应用）
sql
CREATE TABLE lowcode_app (    id BIGSERIAL PRIMARY KEY,    tenant_id BIGINT NOT NULL,    app_name VARCHAR(100) NOT NULL,    app_type VARCHAR(50),    page_json JSONB,  -- 页面配置    data_model JSONB,  -- 数据模型    status VARCHAR(20) DEFAULT 'draft',    created_at TIMESTAMP,    updated_at TIMESTAMP);
5.7 智能体表
sql
CREATE TABLE agent_definition (    id BIGSERIAL PRIMARY KEY,    tenant_id VARCHAR(64) NOT NULL,    name VARCHAR(200) NOT NULL,    description TEXT,    status VARCHAR(20) DEFAULT 'draft',    config JSONB NOT NULL,    bpmn_xml TEXT,    created_at TIMESTAMP);

七、授权许可系统设计
7.1 许可类型
类型	说明	限制
TRIAL	试用版	30天，10用户，1应用
PROFESSIONAL	专业版	按年，500用户，20应用
ENTERPRISE	企业版	永久，不限用户，不限应用
OEM	OEM版	永久，可贴牌分发
7.2 许可约束项
约束项	说明
maxTenants	最大租户数
maxUsersPerTenant	每租户最大用户数
maxAppsPerTenant	每租户最大应用数
expireDate	有效期
bindServers	绑定服务器（防盗版）
features	功能开关（源码导出、AI生成等）
7.3 License文件格式
json
{  "companyName": "XX科技有限公司",  "companyCode": "XX2024001",  "licenseType": "ENTERPRISE",  "issueDate": "2026-01-01",  "expireDate": "2099-12-31",  "maxTenants": 9999,  "maxUsersPerTenant": -1,  "maxAppsPerTenant": -1,  "bindServers": ["server-01", "server-02"],  "features": ["source_export", "ai_generate", "workflow"],  "signature": "RSA_SIGNATURE"}
